# CLAUDE.md - AI 协作操作手册

> 本文档面向 **0 代码基础用户**，指导如何与 AI 助手协作完成开发任务。

---

## 🎯 核心理念：自然语言编程

**你不需要会写代码，只需要会说清楚需求。**

AI 助手是你的「技术翻译官」：

- 你把想法用**自然语言**描述出来
- AI 帮你翻译成**代码**实现
- 你只需要检查结果是否符合预期

---

## 📝 与 AI 协作的标准流程

### Step 1: 描述需求（最重要）

**✅ 好的需求描述示例：**

```
我想在登录页面增加一个"记住我"的功能，
用户勾选后，下次访问自动填充用户名。
要求：
1. 默认不勾选
2. 记住的用户名保存在浏览器本地
3. 界面样式和现有保持一致
```

**❌ 不好的需求描述：**

```
做个记住密码功能。
```

**需求描述公式：**

```
我想 [做什么功能]
具体需求：
1. [要求 1]
2. [要求 2]
3. [要求 3]
参考：[如果有参考图/其他页面，可以说明]
```

### Step 2: AI 生成方案

AI 会回复：

- 技术实现思路（可选读）
- 具体代码变更
- 需要你确认的事项

### Step 3: 你确认或调整

**确认方式：**

- "可以，就这样实现"
- "第 2 点需要调整，应该是..."
- "我不理解 XXX，能解释一下吗？"

### Step 4: AI 执行并验证

AI 执行代码修改后，你可以：

- 在浏览器查看效果
- 告诉 AI "运行正常" 或 "有问题：..."

## 🗣️ 常用指令模板

```

1.
2.
3.
```

```
我想修改 [页面/功能名称]：
- 当前问题：
- 期望效果：
- 参考（如有）：
```

### Bug 修复

```
我发现一个 Bug：
- 现象：
- 复现步骤：
- 期望结果：
- 实际结果：
```

### 代码解释

```
请解释 [文件路径] 这段代码的作用：
```

（粘贴代码片段）

## 🔧 小白维护指南

### 如何添加新页面

**告诉 AI：**

```
我想添加一个新页面，路径是 /settings/profile，
页面标题是"个人设置"，包含：
1. 头像上传区域
2. 昵称输入框
3. 保存按钮
```

**AI 会帮你：**

1. 创建页面文件
2. 添加路由
3. 实现基础样式
4. 更新导航菜单

### 如何修改样式

**告诉 AI：**

```
把登录页面的按钮颜色从蓝色改成绿色，
具体色值参考 #22c55e
```

### 如何添加数据库字段

**告诉 AI：**

```
我想在「项目」表里增加一个字段：
- 字段名：description（项目描述）
- 类型：文本
- 可选填
请帮我更新数据库和相关的页面。
```

## ⚠️ 重要约定

### 1. 先问后做

**复杂功能**必须先让 AI 出方案，你确认后再执行。

```
用户：我想做一个数据导入功能，支持 Excel 和 CSV
AI：[输出详细方案]
用户：确认，开始实现
AI：[执行代码]
```

### 2. 小步快跑

**不要一次提太多需求**，分批次实现：

- ✅ 先做基础功能
- ✅ 验证可用
- ✅ 再添加增强功能

### 3. 及时验证

AI 每完成一个功能，**立即在浏览器验证**：

- 符合预期 → 告诉 AI "没问题"
- 有问题 → 截图或描述现象

### 4. 保持同步

修改代码前，**先询问当前状态**：

```
我想修改用户登录功能，请先告诉我：
1. 当前登录功能是怎么实现的？
2. 涉及哪些文件？
```

## 📚 技术概念速查（非必读）

### 前端

- **组件** = 页面的「积木块」（如按钮、输入框）
- **路由** = 网址和页面的对应关系
- **状态** = 页面上的数据（如输入框里的文字）

### 后端

- **API** = 前端和后端之间的「对话接口」
- **数据库** = 存放数据的「仓库」
- **ORM** = 方便操作数据库的工具

### AI

- **Prompt** = 你给 AI 的指令/问题
- **Token** = AI 处理文字的基本单位
- **模型** = Kimi/GPT 等 AI 大脑

## 🆘 遇到问题怎么办

| 问题类型     | 处理方式                               |
| ------------ | -------------------------------------- |
| 代码报错     | 复制错误信息给 AI                      |
| 界面显示不对 | 截图 + 描述期望效果                    |
| 功能不工作   | 描述操作步骤 + 实际结果                |
| 不知道怎么做 | 直接问 AI "我想做... 应该从哪里开始？" |

## ✅ 功能开发 Checklist

新增功能前，让 AI 确认：

- [ ] 是否涉及数据库变更？
- [ ] 是否需要新增 API 接口？
- [ ] 是否涉及权限控制？
- [ ] 是否需要添加测试？
- [ ] 是否需要更新文档？

## 🔄 项目状态同步

每次对话开始时，可以问：

```
请告诉我当前项目的状态：
1. 最近一次修改了什么？
2. 接下来计划做什么？
3. 有什么需要我确认的吗？
```
