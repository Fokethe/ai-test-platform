## 1. 新功能开发 Prompt

### 使用场景

需要开发全新的功能模块或页面时使用

### Prompt 模板

```markdown
## 角色设定

你是一位资深全栈开发工程师，精通 Next.js + React + TypeScript + Prisma 技术栈。

## 当前任务

我需要开发 [具体功能描述]。

## 上下文信息

- 项目路径: ai-test-platform/my-app
- 技术栈: Next.js 16 + React 19 + TypeScript 5 + TailwindCSS 4 + Prisma 6 + shadcn/ui
- 数据库: SQLite (开发) / PostgreSQL (生产)
- 认证: NextAuth.js v4

## 已有相关代码

[提供相关的现有代码片段或文件路径]

## 具体需求

1. [需求点1]
2. [需求点2]
3. [需求点3]

## 验收标准

- [ ] 功能实现完整
- [ ] 类型安全 (TypeScript)
- [ ] 错误处理完善
- [ ] 响应式设计
- [ ] 代码符合项目规范

## 额外要求

- 如果需要安装新依赖，请提前告知
- 如果需要数据库变更，请提供迁移脚本
- 如果涉及API变更，请同步更新类型定义

请开始开发，完成后简要说明所做的修改。
```

### 示例

```markdown
## 角色设定

你是一位资深全栈开发工程师...

## 当前任务

实现测试计划管理功能，允许用户创建、编辑、删除测试计划。

## 具体需求

1. 创建测试计划，包含名称、描述、时间范围
2. 查看测试计划列表和详情
3. 编辑测试计划信息
4. 删除测试计划
5. 将测试用例关联到测试计划

## 验收标准

- [ ] 可以创建测试计划并保存到数据库
- [ ] 列表支持分页和搜索
- [ ] 编辑后数据正确更新
- [ ] 删除有二次确认
- [ ] 可以将用例添加到计划
```

````
```markdown
````

````
对 [模块/功能名称] 进行全面的手动功能测试。

#### 包含功能
- [功能点1]
- [功能点2]
- [功能点3]

#### 不包含功能
- [不在本次测试范围的功能]

### 测试类型
☐ 冒烟测试 (核心功能快速验证)
☐ 全面功能测试 (所有功能点)
☐ 回归测试 (验证修复的问题)
☐ 验收测试 (发布前最终验证)

### 测试环境
- 应用版本: [commit hash 或版本号]
- 部署环境: [开发/测试]
- 浏览器: [Chrome 120 / Edge 120 / Firefox 121]
- 操作系统: [Windows 11 / macOS 14]
- 测试账号: [账号/密码]
- 屏幕分辨率: [1920x1080]

### 输出要求
请提供以下交付物：

☐ 1. 测试用例文档
   - 按模块组织的测试用例
   - 每个用例包含：步骤、预期结果、实际结果、状态

☐ 2. 测试执行记录
   - 执行日期和时间
   - 执行人
   - 每个用例的执行结果 (PASS/FAIL/N/A)

☐ 3. 问题报告
   - 发现的所有问题
   - 按严重程度分类 (P0/P1/P2/P3)
   - 包含复现步骤和截图

☐ 4. 测试报告
   - 测试总结
   - 统计信息 (通过率、问题数)
   - 质量评估
   - 发布建议

### 测试重点
请特别关注以下方面：
1. 边界条件处理
2. 异常流程
3. 数据一致性
````

## 9. 视觉还原 Prompt（截图转代码）

### Prompt 模板

```markdown
用 React + Tailwind 精确还原这个界面。

要求：

1. 像素级还原：配色、间距、字体大小、圆角
2. 布局：使用 Flex/Grid，禁止绝对定位（除非是悬浮按钮）
3. 响应式：这是桌面端截图，自动推导移动端布局（折叠/堆叠）
4. 交互：所有按钮添加 hover/active 状态
5. 组件化：拆分为 Header/Content/Sidebar 等逻辑组件

先输出结构分析（这是什么页面，有哪些模块），再输出代码。
```
