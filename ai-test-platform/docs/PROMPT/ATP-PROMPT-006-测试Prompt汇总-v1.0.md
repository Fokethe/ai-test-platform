# Vibe Coding Prompt - 测试模式

## 全面测试 Prompt

```markdown
## 角色设定
你是一位资深测试开发工程师，负责制定测试计划和执行测试。

## 测试目标
对 [模块/功能] 进行全面测试，覆盖率要求 100%。

## 测试范围
- [ ] 单元测试
- [ ] 集成测试
- [ ] 功能测试
- [ ] 接口测试
- [ ] UI 测试

## 相关代码
[提供被测试的代码]

## 输出要求
1. 测试计划文档 (docs/test/)
2. 测试用例文档 (按规范模板)
3. 手动验证清单
4. 问题报告 (如发现问题)

请按照项目测试规范执行。
```

---

## 快速测试 Prompt

```markdown
帮我测试 [功能/模块]。

测试内容:
1. [测试点1]
2. [测试点2]
3. [测试点3]

相关代码:
```typescript
[代码片段]
```

请:
1. 分析潜在问题
2. 编写测试用例
3. 如有问题，提供修复方案
```

## 回归测试 Prompt

```markdown
## 任务
对 [模块] 进行回归测试。

## 变更内容
[描述本次代码变更]

## 影响范围
[可能影响的功能]

## 回归重点
1. [重点1]
2. [重点2]
3. [重点3]

请生成回归测试用例并执行验证。
```

## Bug 验证 Prompt

```markdown
## Bug 描述
[Bug标题和描述]

## 修复方案
[开发人员提供的修复说明]

## 验证步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 预期结果
[验证通过的标准]

请验证 Bug 是否已修复，并更新问题报告状态。
```

---

## 测试文档生成 Prompt

```markdown
请为 [模块] 生成完整的测试文档。

包含:
1. 测试计划 (按 ATP-PLAN-XXX 规范)
2. 测试用例 (按 TC-XXX 规范，至少10个)
3. 验证清单 (可手动填写版本)
4. 测试报告模板

参考文档: docs/test/standards/
```

## 自动化测试 Prompt

```markdown
## 任务

- [API端点列表]

## 工具选择

☐ Postman 集合

## 输出

- 执行说明
- 预期结果

请编写自动化测试代码。
