# DEV-001: 新功能开发

## 使用场景
需要开发全新的功能模块或页面时使用

## Prompt 模板

```markdown
## 新功能开发任务

### 功能需求
我需要实现 [功能名称]。

具体需求如下：
1. [需求点1]
2. [需求点2]
3. [需求点3]

### 业务背景
这个功能是为了解决 [问题描述]。
用户使用场景是 [场景描述]。

### 验收标准 (必须满足)
- [ ] 功能点1: [具体标准]
- [ ] 功能点2: [具体标准]
- [ ] 功能点3: [具体标准]

### 技术约束
- 前端框架: Next.js 16.1.6 + React 19.2.3 + TypeScript 5
- UI组件库: shadcn/ui + TailwindCSS 4
- 数据库: Prisma 6.6.0 + SQLite (开发) / PostgreSQL (生产)
- 认证: NextAuth.js 4.24.13
- AI服务: Kimi API (OpenAI SDK格式)

### UI/UX要求
☐ 响应式设计 (支持移动端)  
☐ 深色模式支持  
☐ 加载状态  
☐ 错误提示友好  
☐ 无障碍支持

### 参考资源
- 类似功能参考: [如有]
- 设计稿: [Figma/图片链接]
- 技术文档: [相关文档]

### 数据模型
```typescript
// 如果需要新表，请描述模型
model Example {
  id        String   @id @default(cuid())
  name      String
  createdAt DateTime @default(now())
}
```

### API设计 (如需要)
- GET /api/examples - 获取列表
- POST /api/examples - 创建
- PATCH /api/examples/:id - 更新
- DELETE /api/examples/:id - 删除

### 优先级
☐ P0 - 阻塞其他功能  
☐ P1 - 重要功能  
☑ P2 - 一般功能

### 期望完成时间
[日期，如: 本周五]

### 备注
[其他需要说明的事项]

---

请按以下步骤执行：

1. **需求分析**
   - 理解需求背景和目标
   - 识别潜在问题和风险
   - 提出澄清问题（如有）

2. **技术方案设计**
   - 设计数据模型（如需要）
   - 设计API接口（如需要）
   - 设计组件结构
   - 选择合适的技术方案

3. **开发实施**
   - 创建/修改数据库模型
   - 开发API接口
   - 开发前端页面/组件
   - 添加错误处理
   - 添加加载状态

4. **测试**
   - 编写测试用例
   - 执行功能测试
   - 验证边界条件

5. **文档**
   - 更新API文档
   - 编写使用说明
   - 更新CHANGELOG

请开始执行，每一步完成后向我汇报进展。
```

## 示例

```markdown
## 新功能开发任务

### 功能需求
实现测试计划管理功能，允许用户创建、编辑、删除测试计划。

具体需求：
1. 创建测试计划，包含名称、描述、时间范围
2. 查看测试计划列表和详情
3. 编辑测试计划信息
4. 删除测试计划
5. 将测试用例关联到测试计划

### 业务背景
目前测试用例分散管理，需要一个计划维度来组织测试活动。

### 验收标准
- [ ] 可以创建测试计划并保存到数据库
- [ ] 列表支持分页和搜索
- [ ] 编辑后数据正确更新
- [ ] 删除有二次确认
- [ ] 可以将用例添加到计划

### 优先级
P1 - 重要功能

### 期望完成时间
3天内
```
