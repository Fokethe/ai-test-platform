# TEST-001: 手动功能测试

## 使用场景
需要人工对系统功能进行全面测试时使用

## Prompt 模板

```markdown
## 手动功能测试任务

### 测试目标
对 [模块/功能名称] 进行全面的手动功能测试。

### 测试范围
#### 包含功能
- [功能点1]
- [功能点2]
- [功能点3]

#### 不包含功能
- [不在本次测试范围的功能]

### 测试类型
☐ 冒烟测试 (核心功能快速验证)  
☐ 全面功能测试 (所有功能点)  
☐ 回归测试 (验证修复的问题)  
☐ 验收测试 (发布前最终验证)

### 测试环境
- 应用版本: [commit hash 或版本号]
- 部署环境: [开发/测试]
- 浏览器: [Chrome 120 / Edge 120 / Firefox 121]
- 操作系统: [Windows 11 / macOS 14]
- 测试账号: [账号/密码]
- 屏幕分辨率: [1920x1080]

### 已知限制
- [已知但未修复的问题]
- [需要绕过的步骤]

### 前置条件
1. 服务已启动: http://localhost:3000
2. 数据库已重置并 seeded
3. 已登录测试账号

### 输出要求
请提供以下交付物：

☐ 1. 测试用例文档  
   - 按模块组织的测试用例
   - 每个用例包含：步骤、预期结果、实际结果、状态

☐ 2. 测试执行记录  
   - 执行日期和时间
   - 执行人
   - 每个用例的执行结果 (PASS/FAIL/N/A)

☐ 3. 问题报告  
   - 发现的所有问题
   - 按严重程度分类 (P0/P1/P2/P3)
   - 包含复现步骤和截图

☐ 4. 测试报告  
   - 测试总结
   - 统计信息 (通过率、问题数)
   - 质量评估
   - 发布建议

### 测试重点
请特别关注以下方面：
1. 边界条件处理
2. 异常流程
3. 数据一致性
4. UI/UX细节
5. 响应式布局

### 参考文档
- 需求文档: [链接]
- 设计稿: [链接]
- 之前的测试报告: [链接]

---

请按以下步骤执行：

1. **测试计划制定**
   - 分析测试范围
   - 识别测试重点
   - 制定测试策略

2. **测试用例设计**
   - 按模块编写测试用例
   - 覆盖正常/异常/边界场景
   - 确保可追溯性

3. **测试执行**
   - 按优先级执行用例
   - 记录实际结果
   - 截图记录问题

4. **问题管理**
   - 分类和定级问题
   - 跟踪问题修复
   - 验证修复结果

5. **报告编写**
   - 汇总测试结果
   - 评估产品质量
   - 给出发布建议

请开始制定详细的测试计划，并创建可填写的验证清单供我执行。
```

## 示例

```markdown
## 手动功能测试任务

### 测试目标
对 AI Test Platform 的测试套件功能进行全面测试。

### 测试范围
包含：测试套件CRUD、添加用例到套件、执行套件、查看执行结果

### 测试类型
全面功能测试

### 测试环境
- 版本: v0.1.0
- 浏览器: Chrome 120
- 系统: Windows 11
- 账号: demo@example.com / password123

### 输出要求
1. 详细的测试用例文档
2. Excel格式的执行清单
3. 问题报告
4. 测试总结报告

### 测试重点
- 套件执行的稳定性
- 大量用例时的性能
- 执行中断的处理
```

## 输出预期

AI 应该输出：
1. 测试计划概述
2. 详细的测试用例集 (按模块)
3. 可填写的验证清单
4. 问题报告模板
5. 测试报告模板
