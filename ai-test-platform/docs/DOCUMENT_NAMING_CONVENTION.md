# AI Test Platform 文档命名规范

> 本文档定义项目所有文档的统一命名规范

---

## 命名格式

```
ATP-[类型]-[序号]-[中文描述]-v[版本号].md
```

### 组成部分说明

| 部分 | 格式 | 示例 |
|------|------|------|
| ATP | 项目代号（固定） | ATP |
| 类型 | 3-4位大写字母 | PRD, PLAN, STD |
| 序号 | 3位数字 | 001, 002 |
| 中文描述 | 简明中文 | 产品需求文档, 测试计划 |
| 版本号 | v数字.数字 | v1.0, v2.1 |

---

## 文档类型代码

| 代码 | 含义 | 用途 |
|------|------|------|
| **PRD** | Product Requirements Document | 产品需求文档 |
| **PLAN** | Plan | 各类计划文档 |
| **STD** | Standard | 规范标准文档 |
| **TC** | Test Case | 测试用例文档 |
| **REP** | Report | 报告类文档 |
| **PROG** | Progress | 进度追踪文档 |
| **GUIDE** | Guide | 指南类文档 |
| **PROMPT** | Prompt | AI Prompt文档 |
| **ARCH** | Architecture | 架构文档 |
| **AUX** | Auxiliary | 辅助文档 |

## 版本号规则

| 版本变化 | 说明 | 示例 |
|----------|------|------|
| v1.0 → v1.1 | 小修改、错误修正 | v1.0 → v1.1 |
| v1.1 → v2.0 | 重大更新、结构调整 | v1.1 → v2.0 |

---

## 创建新文档流程

1. 确定文档类型，选择对应代码
2. 查看该类型现有文档，确定序号（递增）
3. 编写简明中文描述
4. 从 v1.0 开始版本号
5. 在文档头部添加标准模板

---

## 文档头部模板

```markdown
# [文档标题]

> 文档编号: ATP-[类型]-[序号]  
> 版本: v[版本号]  
> 最后更新: YYYY-MM-DD  
> 维护人: [姓名/角色]  
> 状态: ⏳ 编辑中 / ✅ 已完成 / 🔄 持续更新

---
```
