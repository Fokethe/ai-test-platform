## 2. 严重级别定义

### P0 - 阻塞级 (Critical)
**定义:** 系统崩溃、数据丢失、核心功能完全不可用

**示例:**
- 系统无法启动
- 数据库连接失败
- 用户无法登录
- 核心业务流程中断

**响应时间:** 4小时内必须响应

### P1 - 严重级 (Major)
**定义:** 主要功能失效，但有替代方案

**示例:**
- 工作空间无法创建
- 测试用例无法保存
- 报告无法生成
- 性能严重下降 (>10秒)

**响应时间:** 24小时内必须响应

### P2 - 一般级 (Minor)
**定义:** 非核心功能问题，不影响主要流程

**示例:**
- 页面显示异常
- 提示信息错误
- 操作繁琐
- 性能轻微下降

**响应时间:** 72小时内响应

### P3 - 轻微级 (Trivial)
**定义:** 界面/体验问题，不影响功能

**示例:**
- 文字拼写错误
- 样式不统一
- 提示不够友好
- 布局微调

**响应时间:** 下次迭代处理

## 3. 状态流转

```
新建 → 已确认 → 修复中 → 待验证 → 已关闭
  ↓       ↓        ↓        ↓
已拒绝   重复     延期     重开
```

| 状态 | 说明 |
|------|------|
| 新建 | 刚提交的问题 |
| 已确认 | 确认是有效问题 |
| 已拒绝 | 不是问题或重复 |
| 修复中 | 开发人员正在修复 |
| 待验证 | 修复完成，等待测试 |
| 已关闭 | 验证通过，关闭 |
| 重开 | 验证不通过，重新打开 |
