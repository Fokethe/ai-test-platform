# 测试用例规范文档 (Test Case Standard)

> 文档编号: ATP-STD-002  
> 版本: v1.0  
> 适用范围: AI Test Platform 项目全周期测试

---

## 1. 测试用例模板

每个测试用例必须包含以下字段：

```markdown
### TC-[编号]: [用例标题]

**基本信息:**
| 字段 | 内容 |
|------|------|
| 用例编号 | TC-XXX |
| 所属模块 | [模块名称] |
| 功能点 | [功能点] |
| 优先级 | P0/P1/P2/P3 |
| 测试类型 | 功能/接口/UI/性能/安全 |
| 创建人 | [姓名] |
| 创建日期 | YYYY-MM-DD |
| 最后更新 | YYYY-MM-DD |

**前置条件:**
1. [条件1]
2. [条件2]

**测试步骤:**
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | [操作1] | [预期1] |
| 2 | [操作2] | [预期2] |
| 3 | [操作3] | [预期3] |

**测试数据:**
- [数据项1]: [值]
- [数据项2]: [值]

**后置条件:**
- [清理操作]

**执行记录:**
| 轮次 | 日期 | 执行人 | 结果 | 备注 |
|------|------|--------|------|------|
| 1 | 2026-02-16 | [姓名] | PASS/FAIL | [备注] |
```

## 2. 用例编号规则

```
TC-[模块代码]-[序号]-[类型]

模块代码:
- AUTH: 认证模块
- WS: 工作空间
- PROJ: 项目
- SYS: 系统
- PAGE: 页面
- TC: 测试用例
- SUITE: 测试套件
- EXEC: 执行
- REPORT: 报告
- USER: 用户设置

类型:
- POS: 正向用例
- NEG: 反向用例
- BDY: 边界值
- UI: UI测试

示例:
TC-AUTH-001-POS: 正常登录
TC-WS-001-NEG: 无权限访问工作空间
```

## 3. 用例设计方法

### 3.1 等价类划分
- 有效等价类: 符合要求的输入
- 无效等价类: 不符合要求的输入

### 3.2 边界值分析
- 最小值
- 最小值+1
- 正常值
- 最大值-1
- 最大值

### 3.3 场景法
- 基本流: 正常业务流程
- 备选流: 分支业务流程
- 异常流: 错误处理流程

### 3.4 状态迁移
- 初始状态
- 中间状态
- 终止状态
- 状态转换条件

---

## 4. 用例评审标准

| 检查项 | 标准 |
|--------|------|
| 完整性 | 覆盖所有功能点和场景 |
| 正确性 | 步骤和预期结果正确 |
| 可执行 | 步骤清晰，可操作 |
| 独立性 | 用例之间无依赖 |
| 可重复 | 重复执行结果一致 |
| 可追溯 | 可追溯到需求和设计 |

---

## 5. 用例维护规范

### 5.1 新增用例
1. 需求变更时
2. 发现新场景时
3. Bug修复后补充回归用例

### 5.2 修改用例
1. 需求变更时
2. 发现用例错误时
3. 执行结果不一致时

### 5.3 废弃用例
1. 功能下线时
2. 用例合并时
3. 用例重复时
